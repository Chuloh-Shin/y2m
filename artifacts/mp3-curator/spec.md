# MP3 Curator — spec.md

## 개요
사용자가 자연어로 취향을 한 줄 입력하면, LLM이 만든 곡 목록을 확인한 뒤 일괄 mp3 다운로드를 시작할 수 있다. 알고 있는 한 곡을 URL로 받는 부수 진입점도 같은 화면에 있다.

## 범위

### 포함
- 취향 한 줄 + 곡 수 N(1–50, 기본 10)을 입력하는 화면
- LLM이 만든 N개의 곡 목록(곡명·아티스트·썸네일·YouTube URL 링크)을 보여주는 확인 화면, 그리고 "다운로드 시작" / "다시 생성" / "입력 수정" 액션
- 곡 목록 항목별 체크박스 — 기본 전부 체크됨. 체크된 곡만 다운로드 대상
- 곡 목록 항목의 YouTube URL 링크 클릭 시 새 탭에서 영상을 열어 미리 들어볼 수 있음
- 단일 YouTube URL을 입력해 한 곡을 받는 부수 입력
- 곡별 카드로 보이는 변환 진행 상태 (대기 / 검색 중 / 다운로드 중 / 변환 중 / 완료 / 실패+사유)
- 일부 곡이 실패해도 나머지는 진행하는 부분 실패 처리
- 빈 입력·범위 밖 곡 수 제출 차단
- LLM 호출 실패 시 사용자에게 노출되는 에러 메시지

### 제외
- **계정·인증·결제** — 본인용 personal tool. 사용자가 1명.
- **다운로드된 파일의 인앱 재생·라이브러리·태그 관리** — OS 플레이어로 충분.
- **음질 옵션 선택** — 기본 설정 하나로 충분.
- **항목별 "이 곡만 다시 생성" 같은 부분 재추천** — UI 복잡도. 마음에 안 드는 곡은 체크 해제로 빼고 전체 "다시 생성"으로 처리.
- **변환 재시도 버튼** — 본인용 minimal. 실패하면 입력부터 다시.
- **변환 진행 상태의 새로고침 복구** — 새로고침은 화면 초기화. 진행된 파일은 다운로드 폴더로 확인.
- **다운로드 폴더 사용자 선택 UI** — 브라우저의 기본 다운로드 동작에 위임.
- **공개 호스팅** — 로컬 `bun dev` 한정. ToS 리스크 회피.

## 시나리오

### 1. 취향 모드 — happy path
- **Given** 입력 화면에 취향 텍스트박스가 비어 있고 곡 수가 `10`으로 표시되어 있다
- **When** 취향 텍스트박스에 `"여성 20대가 좋아하는 최신 댄스곡"`을 입력하고 곡 수를 `5`로 바꾼 뒤 `"곡 목록 만들기"`를 클릭한다
- **Then** 곡 목록 확인 화면으로 전환되고, 5개 항목이 표시된다. 각 항목에는 곡명·아티스트·썸네일 이미지·YouTube URL 링크·체크박스(기본 체크됨)가 있다. `"다운로드 시작"`, `"다시 생성"`, `"입력 수정"` 세 액션이 보인다

성공 기준:
- [ ] 곡 수 = 5 입력 → 확인 화면에 곡 항목이 정확히 5개 표시된다
- [ ] 각 항목에 비어 있지 않은 곡명·아티스트 텍스트, 썸네일 이미지, 외부 링크(YouTube URL)가 표시된다
- [ ] 각 항목의 체크박스가 초기에 체크된 상태다
- [ ] YouTube URL 링크 클릭 → 새 탭에서 해당 영상이 열린다
- [ ] `"다운로드 시작"` 클릭 → 진행 화면으로 전환되고 체크된 곡 수만큼의 카드가 보인다 (이 시나리오에서는 5개)
- [ ] 각 카드가 시간이 지남에 따라 `완료` 또는 `실패` 종결 상태로 이동한다
- [ ] 종결 후 `완료` 상태인 곡 수만큼 mp3가 브라우저의 기본 다운로드 폴더에 저장되어 있다

### 1-a. 취향 모드 — 일부 곡 제외 후 다운로드
- **Given** 곡 목록 확인 화면에 5곡이 모두 체크된 상태로 표시되어 있다
- **When** 2번째와 4번째 항목의 체크박스를 해제하고 `"다운로드 시작"`을 클릭한다
- **Then** 진행 화면으로 전환되어 카드 3개가 표시되고, 해제된 곡은 다운로드 대상에서 빠진다

성공 기준:
- [ ] 5곡 중 2곡 체크 해제 → `"다운로드 시작"` 버튼 라벨·표시가 선택된 곡 수(3개)를 반영한다
- [ ] 진행 화면에 카드가 정확히 3개 표시된다 (체크된 곡만)
- [ ] 모든 카드가 종결 상태에 도달하면 다운로드 폴더에 최대 3개의 mp3가 저장된다 (실패한 곡 제외)
- [ ] 모든 항목의 체크를 해제하면 `"다운로드 시작"`이 비활성화 상태로 보인다

### 2. URL 모드 — happy path
- **Given** 입력 화면 하단의 URL 입력란이 비어 있다
- **When** 유효한 YouTube 영상 URL을 입력하고 `"변환"`을 클릭한다
- **Then** 진행 화면으로 전환되고 곡 카드 1개가 표시되어 상태가 `완료` 또는 `실패`로 이동한다

성공 기준:
- [ ] 유효한 URL → 카드 1개가 표시되고 결국 `완료` 상태가 된다
- [ ] `완료` 상태일 때 mp3 1개가 다운로드 폴더에 저장되어 있다
- [ ] URL 모드는 취향 모드의 곡 목록 확인 단계를 거치지 않는다

### 3. 곡 목록 다시 생성
- **Given** 곡 목록 확인 화면에서 N개의 곡이 표시되어 있다
- **When** `"다시 생성"`을 클릭한다
- **Then** 같은 취향 입력과 곡 수로 새 곡 목록이 생성되어 화면이 갱신된다. 입력 화면으로 돌아가지 않는다

성공 기준:
- [ ] `"다시 생성"` 클릭 후 항목 개수가 여전히 N개로 표시된다
- [ ] 새 목록의 곡명·아티스트 조합이 직전 목록과 동일하지 않다 (최소 한 곡 이상 다르다)

### 4. 입력 수정
- **Given** 곡 목록 확인 화면에서 N개의 곡이 표시되어 있다
- **When** `"입력 수정"`을 클릭한다
- **Then** 입력 화면으로 돌아가고 직전에 입력한 취향 텍스트와 곡 수가 그대로 보존되어 있다

성공 기준:
- [ ] 입력 화면 복귀 후 취향 텍스트박스에 직전 입력값이 그대로 있다
- [ ] 곡 수 입력에 직전 값이 그대로 있다

### 5. 부분 실패
- **Given** 곡 수 N으로 다운로드를 시작했고, 그중 일부 곡이 YouTube에서 검색되지 않거나 변환이 실패하는 상황이다
- **When** 모든 곡이 종결 상태에 도달한다
- **Then** 성공한 곡은 `완료` 카드와 함께 mp3가 다운로드되고, 실패한 곡은 카드에 사유 텍스트(예: `"YouTube에서 못 찾음"`, `"변환 실패"`)가 표시된다. 화면 전체가 중단되지 않는다

성공 기준:
- [ ] 일부 곡이 실패해도 다른 곡의 변환은 정상 진행되어 종결 상태에 도달한다
- [ ] 실패한 카드에 비어 있지 않은 사유 텍스트가 표시된다
- [ ] 다운로드 폴더에는 `완료` 상태인 곡 수만큼의 mp3 파일이 있다

### 6. 입력 검증
- **Given** 입력 화면이 표시된다
- **When** 취향 텍스트박스를 비운 채로 `"곡 목록 만들기"`를 클릭하거나, 곡 수에 `0` 또는 `51`을 입력하려 한다
- **Then** 곡 목록 생성이 시작되지 않고, 입력 화면에 머무른다

성공 기준:
- [ ] 취향이 빈 상태 → `"곡 목록 만들기"` 버튼이 비활성화 상태로 보이거나 클릭해도 다음 화면으로 가지 않는다
- [ ] 곡 수에 `0` 또는 `51`을 입력 → 입력값이 유효 범위 안으로 제한되거나 제출이 막힌다

### 7. LLM 호출 실패
- **Given** 취향과 곡 수가 입력된 상태다
- **When** `"곡 목록 만들기"`를 클릭했으나 LLM 호출이 실패한다
- **Then** 입력 화면을 유지한 채로 사용자에게 곡 목록을 만들 수 없다는 에러 메시지가 표시된다. 입력값은 보존된다

성공 기준:
- [ ] LLM 실패 → 비어 있지 않은 에러 메시지가 입력 화면에 보인다
- [ ] 입력값(취향·곡 수)이 사라지지 않고 그대로 남아 있다

## 불변 규칙

- **동시 실행 제한** — 곡 목록 생성 또는 변환 진행이 진행 중일 때 사용자가 새로운 요청(취향 모드 또는 URL 모드)을 제출할 수 없다. 입력 컨트롤이 비활성화되어 보인다.
- **다운로드 위치** — 성공한 mp3는 사용자가 별도로 폴더를 고르지 않아도 브라우저의 기본 다운로드 폴더에 자동으로 저장된다.
- **새로고침 동작** — 페이지 새로고침 시 화면은 초기 입력 화면으로 돌아가고, 진행 중인 작업의 UI는 복구되지 않는다.
- **입력 언어** — 취향 입력은 한국어·영어를 자유롭게 섞어 쓸 수 있다.

## 의존성

- 외부 LLM API (취향 → 곡 목록 생성)
- YouTube 검색 (곡명·아티스트 → 영상 매칭)
- 영상에서 오디오를 추출해 mp3로 변환하는 로컬 도구

## 미결정 항목

- **M1 probe 결과의 사용처**: 체크박스 편집이 포함되면서 probe가 70% 미만이어도 사용자가 직접 제거 가능해졌다. 다만 probe 정확도가 낮으면 사용자가 매번 N곡을 모두 들어봐야 하는 부담이 커진다. probe 결과에 따라 (a) 체크박스 기본값을 체크 해제(opt-in)로 두는 옵션, (b) LLM 호출 시 곡 수를 의도적으로 더 많이 뽑아 사용자가 고르게 하는 옵션 등을 추가 검토할 수 있다.

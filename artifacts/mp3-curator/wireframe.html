Notes

첫 진입 화면. 두 진입점(취향 / URL)이 한 화면에 공존한다.

  • 트리거: 앱 시작 · 새로고침 · 곡 목록 화면에서 "입력 수정" · 진행 화면에서 "새 입력으로"
  • 인터랙션: "곡 목록 만들기" 클릭 → 곡 목록 확인 화면 / "변환" 클릭 → 진행 화면 (URL 단일 카드)
  • 전환: 곡 목록 생성 성공 → screen-1 / URL 변환 시작 → screen-2

Notes

LLM이 만든 N곡 목록을 사용자가 검토하는 단계. 각 항목의 썸네일·미리 듣기 링크로 매칭이 맞는지 확인하고, 빼고 싶은 곡은 체크 해제로 제외한다.

  • 트리거: 입력 화면에서 "곡 목록 만들기" 클릭 후 LLM 호출 성공
  • 인터랙션: 체크박스 토글 (다운로드 대상에서 제외/포함) · "미리 듣기" 클릭 (새 탭에서 YouTube 영상) · "다운로드 시작" · "다시 생성" · "입력 수정"
  • 전환: "다운로드 시작" → screen-2 (체크된 곡만) / "다시 생성" → 화면 유지, 목록만 갱신 / "입력 수정" → screen-0 (입력값 보존)

  • 인터랙션: 카드의 라벨은 URL 또는 yt-dlp가 추출한 영상 제목으로 표시
  • 전환: 완료/실패 종결 후 "새 입력으로" 활성

Notes

입력 검증 실패 상태. 취향 비어 있음 → 안내 텍스트 + "곡 목록 만들기" 비활성. 곡 수는 범위 안으로 자동 제한되어 50으로 클램프됨(예: 사용자가 51 입력 시).

  • 트리거: 사용자가 취향을 비운 채 제출 시도 / 곡 수에 0·51 입력
  • 인터랙션: 취향 입력 시작하면 안내 텍스트 사라지고 버튼 활성화. 곡 수는 1–50 범위 안에서만 변경 가능
  • 전환: 입력 충족 → screen-1로 진행
